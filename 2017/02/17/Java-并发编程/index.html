Java 并发编程知识整理

大致整理下Java并发编程相关的知识点。

一本书可以越读越厚，也可以越读越薄。

概念

  1. 线程安全

    当多个线程访问某个类时，不管按此用何种调度方式，并且在主调代码中不需要任何额外的同步或者协同，这个类都能表现出正确的行为，那么就称这个类是线程安全的。

  2. 可见性

    一个线程对内存的修改可以被其他线程感知。

  3. 竞态条件

    在并发编程中，由于不恰当的执行时序而出现不正确的结果的情况称为为竞态条件（或者说对线程的执行顺序敏感，不同的执行顺序会出现不同的值，从而出现不可预知的情况）。

基础构建模块

jdk中封装了一些多线程工具类，供开发者开发线程安全的系统。

这些工具包括：

  • 同步容器：所有的操作都是同步(synchronized)的
  • 并发容器：提供了高效的线程安全的符合操作API
  • 同步工具：用于线程之间的协调操作

同步容器类

  • Vector
  • HashTable

并发容器

  • CurrentHashMap 并发HashMap
  • CopyOnWriteArrayList
  • Queue
    • ConcurrentLinkedQueue 并发队列链表实现
    • BlockingQueue 阻塞队列，可用于构建消费者生产者模式（take/put）

同步工具

  • CountDownLatch 闭锁，等待某一组条件发生(countDown/await)
  • Semaphore 信号量，空着操作某个资源的线程数量(acquire/release)
  • CyclicBarrier 栅栏，所有线程到达同一位置，可重置（await/reset）

并发任务执行

在并发执行任务的场景中，通常需要创建大量的线程去并发执行这些任务。无限制创建线程可能带来的问题：

  • 线程生命周期的开销非常高
  • 资源消耗
    活跃的线程会消耗系统资源，尤其是内存。如果可运行的线程数量多于可用处理器的数量，那么有些线程会闲置。大量的空闲线程会占用许多内存，给垃圾收集器带来压力，并且大量线程竞争CPU资源时还将产生其他的性能开销。此时无限制的的创建线程反而会降低性能。
  • 稳定性
    无限制的创建线程可能导致OOM。

Executor框架与线程池

为了避免无限制创建线程所带来的问题，Java提供了Executor框架，来并发执行异步任务。

Executor允许你指定自己的任务执行策略，包括：

  • 在什么（What）线程中执行任务？（ThreadFactory）
  • 任务按照什么（What）顺序（FIFO、LIFO、优先级）执行？
  • 在队列中有多少个（How Many）任务在等待执行？
  • 如果系统由于过载而需要拒绝一个任务，那么应该选择哪一个（Which）任务

可以通过Executors的静态方法，创建不同策略的Executor

任务取消与关闭

通常会遇到提前结束正在执行的任务，例如用户取消操作，应用提前关闭等。Java没有提供任何机制来安全的关闭线程。
Thread.stop与Thread.suspend提供了这样的机制，但却有缺陷。Thread.stop会抛出一个ThreadDeath错误，导致当前线程被立即停止，当前线程持有的锁所保护的状态变量此时可能处于不一致的状态，而对其他线程可见，造成不可预知的线程安全问题。Thread.suspend与Thread.resume配合使用时，很容易造成死锁，Thread.suspend的线程被终止前如果持有一个锁，而调用Thread.resume试图恢复该线程的线程如果正在尝试获取这个锁，那么将发生死锁。

可以使用线程的中断机制来实现线程的优雅停止。
